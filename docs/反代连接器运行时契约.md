# 反代连接器运行时契约

文档版本：v1.0
编写日期：2026-06-09
适用范围：`gen2api/media2api` 调用 Gemini、Veo、Kling、Jimeng、Grok、Midjourney、Pollinations、ComfyUI、Wan、Luma、Runway 等图片/视频生成反代 sidecar。

## 1. 目标

主平台负责统一 API、Web Cookie/session 与 Agent Provider 资源治理、账号池、租约、计费、资产存储和验收；外部 connector/sidecar 只是可选执行层，用来承载网页自动化、任务轮询、CLI wrapper、MCP-to-HTTP 或远程 agent runtime。新的通用连接器运行时把两件事分开：

1. 主平台到 sidecar 的服务级鉴权：通过 provider `base_config.credential_ref` 或 account secret 解析后放入 sidecar 授权头；历史 `api_key_ref` 只作为旧配置兼容字段。
2. sidecar 到上游平台的账号资源引用：通过 account `credential_ref` 与 `resource_profile` 传给 sidecar；主平台可以加密托管 Web Cookie/session 或 Agent Provider profile，但不会长期明文展开。

## 2. 管理诊断接口

```powershell
curl -H "Authorization: Bearer dev-admin-key" `
  http://127.0.0.1:8080/v1/admin/providers/gemini/connector-runtime
```

返回内容包括：

| 字段 | 说明 |
| --- | --- |
| `runtime_contract.base_url` | sidecar 基础地址 |
| `runtime_contract.endpoints` | 按 operation 推断或配置的提交端点 |
| `runtime_contract.poll_endpoint` | 异步任务轮询端点 |
| `runtime_contract.output_paths` | 结果提取路径 |
| `auth_contract.header_names` | 主平台转发给 sidecar 的标准账号引用头 |
| `accounts[].connector_forwarding` | 每个账号的引用转发预览 |

## 3. 请求头契约

当 account `credential_ref` 是安全引用时，主平台默认向 sidecar 转发：

| Header | 示例 | 用途 |
| --- | --- | --- |
| `X-Media2API-Provider-ID` | `gemini` | provider 标识 |
| `X-Media2API-Account-ID` | `acct_gemini_01` | 主平台账号 ID |
| `X-Media2API-Account-Label` | `Gemini production account 01` | 账号标签 |
| `X-Media2API-Credential-Ref` | `agent://providers/gemini/acct_01` | sidecar/agent helper 需要解析的账号授权引用 |
| `X-Media2API-Credential-Type` | `vault` | 引用类型 |
| `X-Media2API-Credential-Reference-Only` | `true` | 表示主平台没有展开上游明文凭据 |

可在 provider `base_config` 中覆盖 header 名称：

```json
{
  "account_header_prefix": "X-Media2API",
  "credential_ref_header": "X-Connector-Account-Ref",
  "credential_type_header": "X-Connector-Account-Type",
  "forward_credential_ref_headers": true
}
```

## 4. Payload 契约

主平台提交图片/视频任务时会把标准账号上下文合并到 `account` 字段：

```json
{
  "operation": "text_to_image",
  "provider_id": "gemini",
  "model": "nano-banana-pro",
  "prompt": "studio product photo",
  "account": {
    "id": "acct_gemini_01",
    "label": "Gemini production account 01",
    "credential_ref": "agent://providers/gemini/acct_01",
    "credential_ref_type": "agent",
    "credential_reference_only": true,
    "supported_operations": ["text_to_image", "image_to_image", "text_to_video"],
    "supported_provider_models": ["nano-banana-pro", "veo-3.1"],
    "region": "global",
    "plan": "pro"
  }
}
```

sidecar 应优先使用 `account.credential_ref` 或 `X-Media2API-Credential-Ref` 找到真实上游授权材料。

## 5. Provider 配置示例

```json
{
  "base_url": "<agent-runtime-url>",
  "credential_ref": "secret://gemini_connector_service_token",
  "api_key_header": "Authorization",
  "endpoints": {
    "text_to_image": "/v1/images/generations",
    "image_to_image": "/v1/images/edits",
    "text_to_video": "/v1/videos/generations",
    "image_to_video": "/v1/videos/generations"
  },
  "poll_endpoint": "/tasks/{task_id}",
  "cancel_endpoint": "/tasks/{task_id}/cancel",
  "quota_endpoint": "/quota",
  "task_id_paths": ["id", "task.uid", "result.task_id"],
  "status_paths": ["status", "task.state", "result.status"],
  "output_paths": ["data", "result.assets", "result.images", "result.videos"]
}
```

## 6. 请求模板

不同开源连接器的请求体通常不完全一致：有的直接接收 OpenAI 风格字段，有的要求 `input` 包装，有的要求 `model_name`、`task` 或 `arguments`。provider `base_config` 可配置 `request_templates`：

```json
{
  "request_templates": {
    "text_to_image": {
      "input": {
        "prompt": "{prompt}",
        "model": "{model}",
        "account_ref": "{account.credential_ref}",
        "account_type": "{account.credential_ref_type}",
        "reference_only": "{account.credential_reference_only}"
      },
      "options": {
        "size": "{size}",
        "quality": "{quality}",
        "n": "{n}"
      },
      "operation": "{operation}",
      "provider_id": "{provider_id}"
    }
  },
  "merge_standard_account_context": true,
  "drop_empty_template_values": true
}
```

占位符支持从标准任务上下文取值：

| 占位符 | 含义 |
| --- | --- |
| `{prompt}` | 用户提示词 |
| `{model}` / `{provider_model}` | 上游 provider model |
| `{operation}` | `text_to_image`、`image_to_video` 等操作 |
| `{account.id}` | 主平台账号 ID |
| `{account.credential_ref}` | sidecar 应解析的账号引用 |
| `{account.credential_ref_type}` | `secret` 或 `agent`；旧值仅用于兼容导入诊断 |
| `{image}` / `{assets}` | 平台输入资产转换后的可访问 URL |

当整个字段值就是一个占位符时，会保留原始类型。例如 `{account.credential_reference_only}` 会渲染为布尔值 `true`，不是字符串 `"true"`。

## 7. 授权资源会话契约

`/v1/admin/authorized-resource-sessions` 是授权资源会话主路径，`/v1/admin/oauth-sessions` 仅保留为历史兼容路径。它可用于 delegated OAuth、Web session assist、CLI 登录助手、Agent Provider profile 导入或 Cookie 刷新。主平台追踪会话、接收安全引用或一次性明文提交，并把完成结果沉淀为 `web_cookie_provider` 或 `agent_provider` 账号资源。

发起会话：

```http
POST /v1/admin/authorized-resource-sessions
Authorization: Bearer dev-admin-key
Content-Type: application/json
```

请求体示例：

`provider_base_url` 只在对应 provider 的 `input_requirements` 包含 runner/sidecar/runtime endpoint 字段时出现；其他平台必须删除该字段，主平台也会拒绝提交。

```json
{
  "provider_id": "gemini",
  "auth_method": "agent_provider_credential",
  "resource_type": "agent_provider",
  "resource_profile": {"agent_runtime_endpoint": "<agent-runtime-url>"},
  "provider_base_url": "<agent-runtime-url>",
  "supported_operations": ["text_to_image", "text_to_video"],
  "supported_provider_models": ["nano-banana-pro", "veo-3.1"]
}
```

主平台会向 sidecar 的 `/oauth/start` 发送：

```json
{
  "session_id": "arsess_xxx",
  "provider_id": "gemini",
  "account_id": "acct_gemini_xxx",
  "auth_method": "agent_provider_credential",
  "resource_type": "agent_provider",
  "resource_profile": {"agent_runtime_endpoint": "<agent-runtime-url>"},
  "callback_url": "/v1/admin/authorized-resource-sessions/arsess_xxx/callback",
  "requested_operations": ["text_to_image", "text_to_video"],
  "requested_provider_models": ["nano-banana-pro", "veo-3.1"]
}
```

sidecar/agent helper 可返回 `authorize_url`、`connector_session_id`，也可以在 start 阶段直接返回 `credential_ref`。如果需要用户登录，用户在 helper 控制的窗口完成登录后有两种完成方式：

1. 运维点击“完成授权并添加账号”，主平台会轮询 helper 的 `/oauth/sessions/{session_id}` 或等价 status endpoint。
2. helper 主动 POST 到 `/v1/admin/authorized-resource-sessions/{session_id}/callback`，使用管理员或服务 API Key 鉴权。

完成 payload 示例：

```json
{
  "status": "completed",
  "account": {
    "id": "acct_gemini_01",
    "label": "Gemini production account 01",
    "credential_ref": "agent://providers/gemini/acct_01",
    "auth_method": "agent_provider_credential",
    "resource_type": "agent_provider",
    "resource_profile": {"agent_runtime_endpoint": "<agent-runtime-url>", "workspace_policy": "isolated"},
    "supported_operations": ["text_to_image", "image_to_image", "text_to_video"],
    "supported_provider_models": ["nano-banana-pro", "veo-3.1"],
    "quota_buckets": [{"type": "credits", "remaining_estimate": 200}],
    "concurrency_limit": 2,
    "region": "global",
    "plan": "pro"
  }
}
```

主平台会兼容识别 `credential_ref`、`account_ref`、`oauth_ref`、`websession_ref`、`agent_ref`、`token_reference`、`subscription_ref`、`vault_ref` 等旧字段，但完成会话后只应沉淀为 `secret://` Web Cookie/session 或 `agent://` Agent Provider 引用。`vault://`、`oauth://`、`websession://`、`subscription://`、`cli://`、`mcp://`、`endpoint://`、`connector://`、`tokenref://`、`env://` 等前缀只能作为兼容导入或执行层来源解析。明文 cookie、session、profile 只能一次性提交并转为 encrypted secret/ref；不支持把密码、验证码处理流程、风控规避脚本或未托管 token 写入账号池。

completion 成功后，平台会自动调用账号 onboarding，把 `resource_type`、`resource_profile`、`supported_operations`、`supported_provider_models`、`quota_buckets`、`concurrency_limit`、`region`、`plan` 写入 `AccountResource`，后续任务通过标准连接器运行时把账号资源引用和脱敏画像转发给 sidecar。

## 8. 结果返回契约

同步结果可直接返回：

```json
{
  "id": "task_123",
  "status": "completed",
  "data": [
    {"image_url": "https://your-authorized-runner.example/media/1.png"},
    {"b64_json": "...", "mime_type": "image/png"}
  ]
}
```

异步结果先返回任务 ID：

```json
{"id": "task_123", "status": "queued"}
```

轮询端点最终返回可提取的 `url`、`image_url`、`video_url`、`media_url`、`download_url`、`b64_json`、`image_base64`、`video_base64` 或 `asset_id`。

## 9. 验证脚本

```powershell
.\.venv\Scripts\python.exe scripts\connector_smoke_test.py
.\.venv\Scripts\python.exe scripts\oauth_session_smoke_test.py
.\.venv\Scripts\python.exe scripts\mixed_media_connector_acceptance_test.py
```

该脚本验证：

1. provider 使用 `secret://` 服务级 token 授权 sidecar。
2. account 使用 `web_cookie_provider` 或 `agent_provider` 资源画像，敏感值以 `secret://` 或 `agent://` 引用转发；旧前缀只在导入阶段兼容解析。
3. sidecar 收到 `X-Media2API-*` 账号引用头。
4. payload `account` 字段包含 `credential_ref_type` 和 `credential_reference_only`。
5. `request_templates` 能把标准任务上下文转换成 sidecar 自定义请求体。
6. 同步结果、异步轮询、自定义结果路径和 quota sync 均可用。
7. 授权资源 completion 和 callback 能把 helper 结果自动落成 Web Cookie 或 Agent Provider 账号。
8. 单个外部 runner/agent 账号可以覆盖 `text_to_image`、`image_edit`、`text_to_video`、`image_to_video` 四个生产必需操作，并通过 preflight、contract suite、account acceptance samples 和外部混合媒体 go-live 快照。
