# media2api 自部署配置指南

本指南对应管理台的「部署向导」和公开 `/setup` 页面，用于在首次部署前完成数据库、Redis、初始管理员、bootstrap API Key 和公开访问地址配置。

## 1. 打开配置向导

- 未登录或首次部署前：访问 `http://你的服务器:端口/setup`
- 已登录管理员后台：进入「部署向导」板块

页面按 sub2api 的 Setup Wizard 思路组织为四步：

1. 数据库：填写 PostgreSQL 或 SQLite 配置。
2. Redis：填写队列、限流和 worker 运行时使用的 Redis。
3. 管理员与 Key：填写初始管理员邮箱、后台密码和 bootstrap API Key。
4. 生成部署：生成 `.env`、启动命令和部署完成检查清单。

## 2. 推荐生产配置

生产环境建议至少配置：

```env
MEDIA2API_ENV=production
DATABASE_URL=postgresql+psycopg2://media2api:change-me@127.0.0.1:5432/media2api
REDIS_URL=redis://127.0.0.1:6379/0
PUBLIC_BASE_URL=http://你的服务器:8080
MEDIA2API_BOOTSTRAP_KEY=替换为强随机值
MEDIA2API_ADMIN_PASSWORD=替换为强密码
MEDIA2API_DEFAULT_USER=admin@example.com
MEDIA2API_INLINE_ASYNC=false
MEDIA2API_WORKER_CONCURRENCY=2
MEDIA2API_SECRET_ENCRYPTION_KEY=替换为强随机值
MEDIA2API_ASSET_SIGNING_SECRET=替换为强随机值
ASSET_DIR=./var/assets
MEDIA2API_ASSET_STORE=local
```

SQLite 只建议本地试用；生产环境应使用 PostgreSQL。

## 3. 启动顺序

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
set -a && source .env && set +a
uvicorn media2api.main:app --host 0.0.0.0 --port 8080
```

另开一个进程或 systemd 服务启动 worker：

```bash
set -a && source .env && set +a
python -m media2api.worker
```

## 4. 部署完成检查

```bash
curl http://你的服务器:8080/health
curl -H "Authorization: Bearer 你的_BOOTSTRAP_KEY" http://你的服务器:8080/v1/runtime
```

然后打开：

- `/admin`：管理后台
- `/setup/status`：只读部署状态
- `/v1/admin/readiness`：管理员就绪检查

## 5. 部署后业务操作路径

1. 进入「授权资源」选择平台，保存 Web Cookie 或 Agent Provider。
2. 进入「账号池」把 `secret://` 或 `agent://` 凭据引用绑定为可路由账号。
3. 进入「模型路由」确认 `gpt-image-2` 等逻辑模型已映射到真实平台模型。
4. 进入「用户与 API Key」创建用户，并给下游系统发放用户 API Key。
5. 运行「账号验收」和「部署检查」，确认真实调用链路可用。

注意：用户 API Key 是下游调用 media2api 的密钥；上游平台账号凭据必须保存在「授权资源/账号池」中，不应直接发给用户。
