# 开源连接器蓝图需求补充

文档版本：v1.0
编写日期：2026-06-09
适用范围：`source-repo/` 开源项目调研结果、connector registry、provider manifest、资源清单源、授权资源会话、生产预检与验收链路。

## 1. 背景

用户的目标不是构建普通 LLM 网关，而是构建一个面向图片、视频生成结果反代输出的媒体版 `sub2api` 平台。平台需要系统性吸收 GitHub 开源社区中的订阅转 API、网页/Web 转 API、CLI/Agent 转 API、MCP、聚合器、自托管图片/视频推理方案，把其中可复用的账号池、资源清单、反代、异步任务和模型能力抽象到 `gen2api/media2api`。

已有能力包括：

- `source-repo/` 本地仓库扫描与项目分类。
- `OpenSourceConnectorProject` 记录 GitHub 项目、平台、模型、操作、鉴权方式和风险。
- provider guide 与 account onboarding plan。
- 兼容 delegated OAuth、Web session assist 与 Agent Provider 授权助手的授权资源会话。
- 账号资源清单的一次性导入、长期同步和生产化预检。
- external connector manifest 安装 provider/account/mapping。

本次补充新增“开源连接器项目蓝图”层，把某个具体 GitHub 仓库从“调研条目”推进成“可应用连接器计划”。

## 2. 新增目标

### R-BP-001 项目级蓝图生成

平台必须支持从 `OpenSourceConnectorProject.id` 生成项目级蓝图，输出：

- 目标 `provider_id`。
- 推断或指定的 `auth_method`。
- 支持的 `operations` 和 provider models。
- external connector manifest payload。
- 可选 account subscription source payload。
- 可选授权资源会话 payload。
- production sync payload。
- preflight、acceptance、manifest-only、subscription source、授权会话 start 等命令。
- license、Web/session、conformance 等风险门禁。

接口：

- `GET /v1/admin/connector-registry/{project_id}/blueprint`

### R-BP-002 项目级蓝图应用

平台必须支持把蓝图应用到当前系统，复用现有安全入口：

- 通过 `ExternalConnectorManifestRequest` 安装 provider/account/mapping。
- 在提供 `subscription_url` 或持久化 `subscription_content` 时创建 `AccountSubscriptionSource`；字段名为兼容旧路径，产品含义是资源清单源。
- 可选运行资源清单源 `production-sync`，默认仍为 dry-run。
- 不执行 `source-repo/` 中任何第三方仓库代码。

接口：

- `POST /v1/admin/connector-registry/{project_id}/blueprint/apply`

### R-BP-003 敏感信息不回显

蓝图响应不得原样回显：

- `credential_value`
- 资源清单内容正文
- 非安全凭据引用

允许回显的用户可见安全引用包括：

- `secret://`
- `agent://`

`env://`、`vault://`、`subscription://`、`oauth://`、`cli://`、`websession://`、`mcp://`、`endpoint://`、`connector://`、`tokenref://` 等旧前缀只能作为兼容导入或执行层来源解析，不应作为新蓝图的平级账号资源输出。

### R-BP-004 Admin 控制台可操作

`/admin` 的“开源连接器”区域必须支持：

- 刷新 `source-repo/` registry。
- 查看 provider guide。
- 生成普通 account onboarding plan。
- 在项目表格中点击“蓝图”自动填入项目 ID。
- 填写 Web Cookie/session 或 Agent Provider profile；`Credential Ref`、资源清单 URL 和执行器地址只按对应项目实际要求出现。
- 生成项目蓝图。
- 应用项目蓝图。

### R-BP-005 交付与验收可见

新增蓝图接口必须出现在：

- `ACCEPTANCE_REQUIRED_ROUTES`
- `/v1/admin/operator-workbench-report`
- `/v1/admin/delivery-package`

交付包中必须提供可复制 curl 命令。

## 3. 操作路径

### 3.1 生成蓝图

```powershell
curl -H "Authorization: Bearer dev-admin-key" `
  "http://127.0.0.1:8080/v1/admin/connector-registry/{project_id}/blueprint?provider_id=gemini&credential_ref=agent://providers/gemini/acct_01"
```

### 3.2 Dry-run 应用

```powershell
curl -X POST `
  -H "Authorization: Bearer dev-admin-key" `
  -H "Content-Type: application/json" `
  http://127.0.0.1:8080/v1/admin/connector-registry/{project_id}/blueprint/apply `
  -d "{
    \"provider_id\":\"gemini\",
    \"credential_ref\":\"agent://providers/gemini/acct_01\",
    \"auth_method\":\"agent_provider_credential\",
    \"resource_type\":\"agent_provider\",
    \"dry_run\":true
  }"
```

### 3.3 正式应用并创建长期资源清单源

```powershell
curl -X POST `
  -H "Authorization: Bearer dev-admin-key" `
  -H "Content-Type: application/json" `
  http://127.0.0.1:8080/v1/admin/connector-registry/{project_id}/blueprint/apply `
  -d "{
    \"provider_id\":\"gemini\",
    \"credential_ref\":\"agent://providers/gemini/acct_01\",
    \"auth_method\":\"agent_provider_credential\",
    \"resource_type\":\"agent_provider\",
    \"subscription_url\":\"https://resource-list.example/gemini/accounts.json\",
    \"create_subscription_source\":true,
    \"dry_run\":false
  }"
```

## 4. 验收标准

- `GET /v1/admin/connector-registry/{project_id}/blueprint` 能返回 manifest、source、授权资源会话、production sync、commands、risk gates。
- `POST /v1/admin/connector-registry/{project_id}/blueprint/apply` 在 `dry_run=true` 时不写数据库。
- `dry_run=false` 且 `apply_manifest=true` 时安装 provider/account/mapping。
- 提供资源清单 URL 且 `create_subscription_source=true` 时创建长期资源清单源。
- 不提供资源清单 URL 或持久化内容时，不创建空资源清单源，并返回 action item。
- 蓝图响应不泄露 `credential_value` 或资源清单内容正文。
- `/admin` 页面包含项目蓝图控件和表格按钮。
- operator workbench 与 delivery package 报告新增路由可见。

## 5. 生产边界

项目蓝图只负责把开源仓库调研结果转化成本平台内的配置与验收计划。它不代表该开源仓库已经可直接生产使用。生产准入仍需满足：

1. Web Cookie/session 或 Agent Provider profile 已按平台实际要求登记；如项目要求外部 runner/sidecar，再确认执行器地址可达。
2. provider health check 通过。
3. capabilities sync 成功。
4. quota sync 或等价预算控制可用。
5. connector contract suite 通过。
6. account acceptance 至少一个真实样本通过。
7. 图片/视频输出已转存为 `MediaAsset`。
8. 高风险 Web/session 类项目已完成人工审核、license 审核和并发限制。
