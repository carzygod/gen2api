# 反代内核开源仓库选型

日期：2026-06-12

本文用于把当前平台从“账号池 + connector 壳”推进到真正可执行的反代内核选型。范围严格按最新约束处理：

- 禁止官方 SDK / 官方公开 API key 路线。
- 只考虑反代、Web OAuth/session、CLI OAuth/session、客户端本地凭据、订阅转 API 类方案。
- 不把第三方反代项目当成平台公开托管服务来内置运营；允许少量 Go/Node 项目作为受控 binary/subprocess，被平台 adapter 调用。
- 优先覆盖 `/v1/images/*` 与 `/v1/videos/*`。
- OpenAI Codex 反代必须和 ChatGPT Web OAuth/session 反代分开。
- Gemini CLI OAuth、Gemini Web OAuth/session、Antigravity 反代必须分开。

## 采用编号规则

你后续只需要按编号回复采用清单，例如：

```text
采用：OAI-WEB-01, OAI-CODEX-01, GROK-01, GEM-WEB-01, KLING-WEB-01
暂不采用：MID-01, QWEN-01
需要再查：RUNWAY-WEB
```

编号保持稳定，不随优先级调整而改名：

| 前缀 | 含义 |
|---|---|
| `OAI-WEB-*` | OpenAI / ChatGPT Web session 反代 |
| `OAI-CODEX-*` | OpenAI Codex CLI/OAuth/session 反代 |
| `GEM-CLI-*` | Gemini CLI OAuth/profile 反代 |
| `GEM-WEB-*` | Gemini Web session 反代 |
| `AG-*` | Antigravity 反代 |
| `GROK-*` | Grok Web/session 反代 |
| `JM-*` | Jimeng / Dreamina / Seedream / Seedance |
| `DOUBAO-WEB-*` | 豆包 Web/session 反代 |
| `KLING-WEB-*` | Kling Web/session 反代 |
| `LUMA-WEB-*` | Luma Web/session 反代 |
| `MID-*` | Midjourney Discord/session 任务通道代理 |
| `QWEN-*` | Qwen Code / Qwen client 反代 |
| `QWEN-AI-*` | `qwen.ai` / `chat.qwen.ai` / `portal.qwen.ai` Web 反代 |
| `QIANWEN-WEB-*` | `qianwen.com` / 通义千问网页版反代 |
| `WENXIN-WEB-*` | 文心一言 / ERNIE Web session 反代 |
| `SELF-*` | 自托管 fallback / worker |
| `EX-*` | 明确排除或仅历史参考 |

## 当前平台兼容判断

| 当前 provider | 当前问题 | 目标拆分 |
|---|---|---|
| `openai_image` | 把 ChatGPT Web 与 Codex CLI/Agent 混在一个资源类型里，用户不知道该导入 cookie、Web session 还是 Codex auth。 | `openai_web_session`、`openai_codex` |
| `gemini` | 把 Gemini CLI OAuth、Gemini Web session、Antigravity 混在一个资源类型里。 | `gemini_cli_oauth`、`gemini_web_session`、`antigravity` |
| `grok` | 已接近 Web/session 反代模型，但需要真实 `/v1/images/*`、`/v1/videos/*` kernel。 | 保留 `grok_web_session` 或 `grok` |
| `jimeng` / `seedream_proxy` | 现有文档偏 Volcengine/Jimeng API key 或 ComfyUI 节点，不符合“禁官方 API”前提；需要优先找 Dreamina/Jimeng Web/内部接口逆向。 | `jimeng_web_session`、`seedream_web_session` |
| 待新增：`doubao_web_session` | 豆包账号与即梦账号日配额不同，不能复用 `jimeng_web_session` 的账号池、并发、额度统计和健康检查。 | `doubao_web_session` |
| 待新增：`qwen_ai_web_session` | `qwen.ai` / `chat.qwen.ai` / `portal.qwen.ai` 是一条独立 Web 反代线，不能和 Qwen Code 或 `qianwen.com` 混用。 | `qwen_ai_web_session` |
| 待新增：`qianwen_web_session` | `qianwen.com` / 通义千问网页版是另一套 Web 产品入口，账号、域名、配额、前端接口都应和 `qwen.ai` 分开。 | `qianwen_web_session` |
| 本轮不新增：`wenxin_web_session` | 用户定型为 `WENXIN：无可用`；文心一言仅保留后续调研线索。 | 暂不进入实现 |
| `kling` | 现有 registry 偏 `KLING_ACCESS_KEY`/MCP，属于官方/半官方 key 路线；应补 Kling Web cookie/session 逆向候选。 | `kling_web_session` |
| `luma` | 现有 registry 偏 Luma API key/MCP；应补 Luma Web cookie/session 逆向候选。 | `luma_web_session` |
| `runway` | 当前候选大多是官方 API 或 useapi/piapi 聚合，和硬性前提冲突。 | 暂列待补逆向 Web 候选 |
| `midjourney` | Discord/MJ channel 代理天然是任务通道反代，和需求方向一致。 | 保留 `midjourney_discord_session` |
| `pollinations` | 如果使用公网 Pollinations API，属于第三方公开聚合 API；如果自托管 worker，可作为非官方 fallback。 | `pollinations_self_hosted` |
| `openrouter_image` / `fal_replicate` | 主要依赖官方/第三方公开 API key，和最新硬性要求冲突。 | 排除或仅保留历史兼容，不进 P0 |

## 集成方式定义

| 方式 | 是否符合 | 用法 |
|---|---:|---|
| 协议重写 | 是，优先 | 参考开源仓库协议逻辑，在 `media2api` 内重写 adapter，统一资产、任务、账号池、审计。 |
| 受控 binary/subprocess | 有条件允许 | 仅对 Go/Node/Python 项目做固定版本、固定 hash、loopback-only 的受控执行器；不暴露其原生后台，不让用户直接操作第三方服务 UI。 |
| 外部用户自管 endpoint | 可作为过渡 | 用户自己部署第三方项目，平台只接入本地/内网 endpoint；但最终仍要收敛到平台自己的 account/resource/job 模型。 |
| 内置托管第三方反代服务 | 不允许 | 不把第三方项目原样作为平台长期托管的公开服务。 |
| 官方 SDK / 官方公开 API | 不允许 | fal、Replicate、OpenRouter、Runway 官方、Luma 官方、Kling 官方、Volcengine 官方等 key-based 方案不进入 P0。 |

## 验收标准

每个选中的内核都必须过下面的验收，未过不得在 dashboard 标为“可用”：

1. 账号导入：只接本人授权的 session/OAuth/profile/credential file/ref；密文入库；日志不输出敏感值。
2. 健康检查：能区分未登录、额度不足、限流、上游格式变化、生成失败。
3. 图片接口：`POST /v1/images/generations` 返回可下载资产；`POST /v1/images/edits` 支持至少单图编辑；资产落 `media_assets`。
4. 视频接口：`POST /v1/videos` 创建异步任务；`GET /v1/videos/{id}` 查询状态；完成后提供 `content` 或 asset URL。
5. 任务模型：所有长任务进入队列，有超时、重试、取消、失败原因。
6. 并发策略：Web/session 类默认账号并发 1；CLI/Agent 类按仓库实测上限配置。
7. 轮换策略：按 provider/account/job type 轮换，不跨 `openai_web_session` 与 `openai_codex`，不跨 `gemini_cli_oauth`、`gemini_web_session`、`antigravity`。
8. 输出归一：无论上游返回 URL、base64、文件路径、Discord CDN、浏览器下载，都统一入平台 asset 存储。
9. UI 呈现：用户只看到“平台、账号类型、状态、额度、支持动作、最近任务”，不暴露上游晦涩字段。
10. 供应链：如果作为 subprocess，必须固定版本/hash、loopback-only、无自动更新、可一键停止、可查看最小运行日志。
